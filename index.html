Lab 1: Introduction to Powershell


    //Get familiar with the different PowerShell environments and run basic commands on your Windows Operating System (Whether it’s Windows Server or Windows Desktop).

  1. In your Amazon Workspace, click the ‘Search’ icon and type "PowerShell".
  2. Choose the top choice "Windows PowerShell" and make it full screen.
  3. Let’s run some commands. Note that we can only type and enter a command one line at a time.
     a. In the "Windows PowerShell Window", type Get-Help and press the Enter/Return key.
     Hint: Get-Help is a PowerShell command that displays help information about PowerShell commands and concepts. PowerShell Help describes PowerShell cmdlets, functions, scripts, and modules, and explains concepts, including the elements of the PowerShell language

    4.  Run:

     b. In the "Windows PowerShell Window", type Get-Alias and press the Enter key.
     Hint: The Get-Alias command provides a list of pre-defined aliases for commands that you can enter in place of the full command names (ex. You can type TNC instead of writing out the Test-NetConnection command, allowing you to save time in the console)

    Run:

  4. Close ‘Windows PowerShell’.


  5. Now let’s compare that to Windows PowerShell ISE!
    In your Amazon Workspace, click the "Search" icon and type "PowerShell". Choose "Windows PowerShell ISE".
  6. In the "Windows PowerShell ISE", we’re going to turn on the script pane.

     a. Click 'View'
     b. Click 'Show Script Pane'
  7. Let’s run some commands. Hint: Note how it displays and interacts differently than Windows PowerShell Console!
      a. In the white box at the top, type Get-Help. Click on the green ‘Play’ button in the task bar to run the command.
       Note: it may take up to 30 seconds for the play button to become available, just be patient!

    Run:

     b. In the white box at the top, delete the Get-Help command and type Get-Alias. Click on the green ‘Play’ button in the task bar to run the command.

    Run:
  8. With Windows PowerShell ISE, we can run multiple lines of commands. to do so,
      a. In the white box at the top, type Get-Help. Then type, Get-Alias
      b. Click on the green 'Play' button in the task bar to run the command.
      c. Scroll up in the bottom blue area and note that both commands ran successfully!

    Run:
  9. Close Windows PowerShell ISE.

    Congratulations! You completed Lab 1!

Lab 2: Elevated Privileges


    //This lab demonstrates how certain commands behave differently depending on user privilege levels. Students run commands in standard and administrator PowerShell sessions and note the difference in results and permissions.

  1. In your Amazon Workspace, open Windows PowerShell.
  2. Let’s run some commands.Note, you’re running these commands as a non-admin.
    a. In the ‘Windows PowerShell Window", type Get-Process and press the Enter key
     The Get-Process command shows all running processes on the machine
    Run:

    b. In the ‘Windows PowerShell Window", type Get-ChildItem and press the Enter key.
     The Get-ChildItem displays the items and child items in one or more specified locations.
    Run:

    c. In the ‘Windows PowerShell Window", type Get-Date and press the Enter key
     We're displaying the current date and time!
    Run:

    d. In the ‘Windows PowerShell Window", type Set-ExecutionPolicy – ExecutionPolicy Unrestricted and press the Enter key.
     Reminder: Although the commands are NOT case sensitive, spaces between commands and parameters must be enforced or you will receive errors when trying to run the command.
    Run:
     Uh-oh! We don't have permissions to do that!
  3. Now let’s try to create a folder with our non-admin PowerShell!
    a. In the ‘Windows PowerShell Window", type New-Item –Path "C:\Program Files\Newfolder" -ItemType Directory and press the Enter key
    Run:
     Uh-oh! We don't have permissions to do that!
  4. Close your Windows PowerShell Session
  5. In order to run these commands that are restricted, we need to open an Admin PowerShell Session.
    a. In your Amazon Workspace, click the ‘Search’ icon and type "PowerShell"
    b. RIGHT CLICK "Windows PowerShell"
    c. Choose "Run as administrator" and click "Yes"
  6. Your Window now reads "Administrator: Windows PowerShell." Let’s run some commands!
    a. In the ‘Administrator: Windows PowerShell" window, type Get-Process and press the Enter key
     Note that the outputs are EXACTLY the same as without running as administrator.
    Run:

    b. In the ‘Windows PowerShell Window", type Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned and press the Enter key
     i. Type in 'y' and press the Enter Key
    Note that there’s no error! We NEED the administrator window for this to work
    Run:

    c. In the ‘Windows PowerShell Window", type New-Item –Path "C:\Program Files\Newfolder" -ItemType Directory and press the Enter key
    Again, note that there’s no error! We NEED the administrator window for this to work
    Run:

    d. Let’s test to make sure the new folder actually got created. In the ‘Windows PowerShell Window", type Test-Path "C:\Program Files\Newfolder" and press the Enter key
    Run:

Lab 4: Your First PowerShell Script


    //What’s the purpose of this lab? Write your first script!

  1. In your Amazon Workspace, open Windows PowerShell.
  2. In your Amazon Workspace, open File Explorer and place it next to your open PowerShell Window. Navigate to your “UserProfile (D:)” drive.
  3. Let's create a directory.
    a. In the ‘Windows PowerShell Window”, type New-Item -Path D:\LabTest -ItemType Directory -Force and press the Enter key.
      Note: When you create the folder in PowerShell, you can SEE the folder be created in the D: Drive!
    Run:
  4. Let's create a text file inside the folder “LabTest” and then add some text to the file.
    a. In the ‘Windows PowerShell Window”, type New-Item -Path D:\LabTest\testfile.txt -ItemType File -Force and press the Enter key.
      Note: When you create the folder in PowerShell, you can SEE the file be created in the LabTest Folder!
    Run:

    b. To write text in the file, type “Hello, PowerShell” | Out-File D:\LabTest\testfile.txt and press the Enter key. If you open the text file, you can see the text written in!
      Hint: the | is Shift + \ (Hold shift and press the backslash)
    Run:
  5. Let's rename the file and move it to a new folder!
    a. In the ‘Windows PowerShell Window”, type Rename-Item -Path D:\LabTest\testfile.txt -NewName newfile.txt -Force and press the Enter key. Verify the new name in the folder!
    Run:

    b. Now let’s move the file to D:\Users. Type Move-Item -Path D:\LabTest\newfile.txt -Destination D:\Users -Force and press the Enter key. Verify the file moved to the folder!
    Run:
  6. To finish, we'll delete the original directory
    a. Type Remove-Item -Path D:\LabTest -Recurse -Force and press the Enter key. Verify the folder has been removed!
    Run:

  7. That was fun! But typing and executing each command separately can get annoying and cumbersome. Let’s use PowerShell ISE to complete multiple steps at once.Close PowerShell and Open PowerShell ISE.
  8. Let’s see how much we can do simultaneously. In PowerShell ISE, we’re going to create a Lab Files folder and in that Lab Files folder create two folders named “Source” and “Destination”. In the same step, we’ll create a test file to use for moving. When done, make sure to click the green play button!
    New-Item -Path "D:\LabFiles" -ItemType Directory
    New-Item -Path "D:\LabFiles\Source" -ItemType Directory
    New-Item -Path "D:\LabFiles\Destination" -ItemType Directory
    Set-Content -Path "D:\LabFiles\Source\ExampleFile.txt” -Value “This is a test file”
  9. In one step, we’ve created folders and a file! The next step is setting up a script to move the file – and we’ll set up a rule to give us a confirmation of a successful move!
    a. Let’s create a new script pane. Click the “New” in the top left corner of the PowerShell ISE Window.

    b. Let’s set up variables that define our Source and Destination.
      Optional: you can ‘comment’ each section to describe what’s happening – just start your line with a #

    c. With our variables defined, let’s set up our script. First thing to check for is if the file exists!
      Hint: if the file does NOT exist, PowerShell will return the message “Source file does not exist”
    if (Test-Path $source) {
        # Code to move file will go here
    } else {
      Write-Host "Error moving source file: $source"}

    d. With our variables defined, our conditional statement written, the only thing left is to actually move the file! Replace the green placeholder text that reads “#Code to move file goes here!” with the below:
    Move-Item – Path $source -Destination $destination -Force
    Write-Host “File moved successfully from $source to $destination”
  10. Let’s test it! Click the green play button in the top bar! You should see the successful message at the bottom of the screen!


    11. Congratulations! You completed Lab 4!
